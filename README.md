# Music Store Data Analysis Using SQL

Music Store Data Analysis using SQL. This project analyzes customer behavior, sales trends, popular genres, artist performance, and country-wise music preferences using SQL queries, joins, CTEs, subqueries, and window functions.

## Project Overview

This project analyzes a Music Store database using SQL to extract meaningful business insights from customer purchases, music genres, artists, albums, and sales transactions. The objective is to demonstrate practical SQL skills by solving real-world business problems using relational database concepts.

---

## Business Problem

The music store generates large volumes of transactional data, but raw data alone does not provide valuable insights. This project aims to analyze customer behavior, identify popular genres and artists, evaluate sales performance, and support data-driven decision-making through SQL queries.

---

## Database Schema

The database consists of the following tables:

* Employee
* Customer
* Invoice
* InvoiceLine
* Artist
* Album
* Track
* Genre
* MediaType
* Playlist
* PlaylistTrack

### Relationship Flow

Customer → Invoice → InvoiceLine → Track → Album → Artist

## Business Questions Solved

1. Who is the senior-most employee based on job title?
2. Which countries have the most invoices?
3. What are the top invoice values?
4. Which city generates the highest revenue?
5. Who is the best customer?
6. Who are the Rock music listeners?
7. Which artists have the most Rock music tracks?
8. Which tracks are longer than the average song length?
9. How much money does each customer spend on artists?
10. What is the most popular music genre in each country?
11. Who is the top customer in each country?
